
# webpack优化
### 缩小文件搜索范围

```js
{
  module: {
    rules: [
      {
        test: '\.js$',
        // 仅对需要处理的文件执行loader，所以限制src
        include: path.resolve(__dirname, 'src'), // 仅
        use: ['babel-loader?cacheDirectory']
      }
    ],
    
    // 不去解析一些大型包，这些包不会依赖和导入其他文件，所有我们可以不去解析它
    noParse: [/react\.min\.js/, /jQuery/]

  },

  resolve: {
    // 引入模块时，默认会在引入文件的当前目录的node_module里面找，找不到就进入上层，如果都是安装在根目录，可以直接指定在根目录寻找
    modules: [path.resolve(__dirname, 'node_modules')],
    // 如果所有依赖都是以main为主要模块时，我们提前指定，加快搜索过程
    mainFields: ['main'],

    // 配置合适的别名，能够快速的找到对应的模块
    alias: {
      react: '...'
    }
  },

  // 自动添加后缀，会影响打包速度，这里保持，最常用到的后缀放在最前面
  extensions: ['js', 'jsx']
}

```
